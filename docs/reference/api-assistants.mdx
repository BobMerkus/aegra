Assistants are configured instances of your graphs. Each assistant has a name, metadata, and configuration that controls how the graph executes.

## Create assistant

<ParamField path="POST" type="/assistants">
  Create a new assistant.
</ParamField>

### Request body

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `graph_id` | `string` | Yes | — | Graph ID from `aegra.json` |
| `assistant_id` | `string` | No | Auto-generated | Client-supplied ID for idempotent creation |
| `name` | `string` | No | Auto-generated | Human-readable name |
| `description` | `string` | No | `null` | Assistant description |
| `config` | `object` | No | `{}` | Graph configuration |
| `context` | `object` | No | `{}` | Graph context |
| `metadata` | `object` | No | `{}` | Searchable metadata |
| `if_exists` | `string` | No | `"error"` | Behavior on duplicate ID: `"error"` or `"do_nothing"` |

### Response

Returns the created `Assistant` object.

```python
from langgraph_sdk import get_client

client = get_client(url="http://localhost:8000")

assistant = await client.assistants.create(
    graph_id="agent",
    name="Customer Support",
    metadata={"department": "support"},
)

print(assistant["assistant_id"])
```

## List assistants

<ParamField path="GET" type="/assistants">
  List all assistants belonging to the authenticated user.
</ParamField>

### Response

| Field | Type | Description |
|-------|------|-------------|
| `assistants` | `list[Assistant]` | List of assistant objects |
| `total` | `integer` | Total count |

```python
result = await client.assistants.list()
for assistant in result["assistants"]:
    print(f"{assistant['name']} ({assistant['assistant_id']})")
```

## Search assistants

<ParamField path="POST" type="/assistants/search">
  Search assistants by name, graph ID, or metadata.
</ParamField>

### Request body

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `string` | `null` | Filter by name |
| `description` | `string` | `null` | Filter by description |
| `graph_id` | `string` | `null` | Filter by graph ID |
| `metadata` | `object` | `{}` | Metadata key-value filters |
| `limit` | `integer` | `20` | Max results (1–100) |
| `offset` | `integer` | `0` | Pagination offset |

### Response

Returns `list[Assistant]`.

```python
results = await client.assistants.search(
    graph_id="agent",
    metadata={"department": "support"},
)
```

## Update assistant

<ParamField path="PATCH" type="/assistants/{assistant_id}">
  Update an assistant's name, description, config, graph, context, or metadata. Creates a new version.
</ParamField>

### Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `assistant_id` | `string` | The assistant ID |

### Request body

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | New name |
| `description` | `string` | New description |
| `config` | `object` | Updated config |
| `graph_id` | `string` | New graph ID |
| `context` | `object` | Updated context |
| `metadata` | `object` | Updated metadata |

All fields are optional. Only provided fields are updated.

### Response

Returns the updated `Assistant` object.

```python
assistant = await client.assistants.update(
    "my-assistant-id",
    name="Updated Name",
    metadata={"department": "engineering"},
)
```

## Get graph

<ParamField path="GET" type="/assistants/{assistant_id}/graph">
  Get the graph structure for visualization.
</ParamField>

### Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `assistant_id` | `string` | The assistant ID |

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `xray` | `boolean \| integer` | `null` | Expand subgraph nodes. `true` expands all, an integer controls depth |

### Response

Returns an object with `nodes` and `edges` for rendering.

```python
# Basic structure
graph = await client.assistants.get_graph("my-assistant-id")

# With subgraph expansion
graph = await client.assistants.get_graph("my-assistant-id", xray=True)

# Control expansion depth
graph = await client.assistants.get_graph("my-assistant-id", xray=2)
```

## Get subgraphs

<ParamField path="GET" type="/assistants/{assistant_id}/subgraphs">
  List subgraphs of an assistant's graph.
</ParamField>

### Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `assistant_id` | `string` | The assistant ID |

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `recurse` | `boolean` | `false` | Recursively list nested subgraphs |
| `namespace` | `string` | `null` | Filter to a specific subgraph namespace |

```python
subgraphs = await client.assistants.get_subgraphs("my-assistant-id")

# Recursive listing
subgraphs = await client.assistants.get_subgraphs("my-assistant-id", recurse=True)
```

## Assistant object

The `Assistant` object returned by all assistant endpoints:

| Field | Type | Description |
|-------|------|-------------|
| `assistant_id` | `string` | Unique identifier |
| `name` | `string` | Human-readable name |
| `description` | `string \| null` | Description |
| `config` | `object` | Graph configuration |
| `context` | `object` | Graph context |
| `graph_id` | `string` | Associated graph ID |
| `user_id` | `string` | Owner identity |
| `version` | `integer` | Version number |
| `metadata` | `object` | Stored metadata |
| `created_at` | `datetime` | Creation timestamp |
| `updated_at` | `datetime` | Last update timestamp |
